# Grafana demo — Linear location

Use this to make sure Grafana demo work goes to the correct Linear workspace/team/project.

## Workspace
- **Workspace:** Ryan — `linear.app/ryan-demos`
- This workspace holds multiple demos, one **team per demo**. Do not put Grafana work in another demo's team (e.g. the Runna-demo / `RUN` team).

## Team (this sets the issue key)
- **Team:** Grafana Demo
- **Issue key prefix:** `GRAF` (e.g. GRAF-1, GRAF-2). All Grafana demo issues use GRAF-.
- When creating an issue, set the team to **Grafana Demo** so it gets a `GRAF-` key. Issue keys are per-team, not per-project.

## Project
- **Project:** Grafana Demo
  - https://linear.app/ryan-demos/project/grafana-demo-5e4884113a2c/overview
- Add new Grafana demo issues to the **Grafana Demo** team and associate them with this project (unless a separate project is explicitly created).

## Where to find the right tickets
- Team board: open the **Grafana Demo** team in the workspace, or filter any view by team = Grafana Demo (key `GRAF`).
- Project view: the **Grafana Demo** project (link above) lists the demo's issues.

## Gotchas
- The workspace slug is `ryan-demos`. Always use that, not any older slug.
- Don't file Grafana issues under the **Runna-demo** team (key `RUN`); that's a different demo.
